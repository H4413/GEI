\subsection{Découpage en sous-systèmes}

Le système peut alors être divisé en trois sous-systèmes principaux :

\begin{enumerate}
\item Le système embarqué
\item Les serveurs (SSH, Sauvegarde, Web, ...)
\item Le service client (logiciel PDA/Smartphone) 
\end {enumerate}

Entre chaque sous-système, nous avons une interface de communication, une API, 
qui se charge de les relier.

\centering
\includegraphics[width=8cm]{\PIXPATH/API}
\end{figure}

\subsection{Les sous-systèmes}
Nous allons décliner la composition de chacun des sous-ensembles ci-dessus afin 
de donner une meilleure granularité à la description.

\subsubsection{Système Embarqué}
\begin{enumerate}

\item Gestion des capteurs
Configuration/maintenance
Suivi des données

\item Gestion de l'énergie
Configuration/maintenance

\item Positionnement
Configuration/maintenance du module GPS
Suivi des données (la position)

\item Communication avec les serveurs
Configuration/maintenance
Suivi des données du système embarqué
Rapports de fonctionnement
Alertes

\item Communication Bluetooth
Configuration/maintenance
Suivi des données des capteurs
Rapports de fonctionnement

\end{enumerate}


\subsubsection{Serveurs}
\begin{enumerate}
\item Serveur de Sauvegarde
Sauvegarde des enregistrements des capteurs dans la base de données

\item Serveur HTTPS (Web)
Configuration des stations
Suivi données provenant des stations

\item Serveur SSH (Communication avec le système embarqué)
Acquisition depuis les stations des enregistrements des capteurs
Configuration des stations
Maintenance logicielle à distance
Suivi des données provenant des stations
\end{enumerate}


\subsubsection{Service client}
\begin{enumerate}
\item Application PDA/Smartphone
Configuration/maintenance
Suivi des données
Rapports de fonctionnement

\item Application Web
Configuration/maintenance
Suivi des données
Rapports de fonctionnement
\end{enumerate}
